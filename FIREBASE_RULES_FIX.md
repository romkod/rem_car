# Виправлення правил Firebase Realtime Database

## Проблема
Зміни не зберігаються на сервері, тому що правила безпеки Firebase не дозволяють запис даних.

## Рішення

### 1. Відкрийте Firebase Console
Перейдіть на: https://console.firebase.google.com/

### 2. Виберіть ваш проект
Проект: `remcar-a23dc`

### 3. Перейдіть до Realtime Database
- У меню зліва виберіть **"Realtime Database"**
- Виберіть вкладку **"Rules"** (Правила)

### 4. Встановіть правила

**Варіант 1: Безпечні правила (рекомендовано)**

```json
{
  "rules": {
    "cars": {
      ".read": true,
      ".write": true,
      "$carId": {
        "categories": {
          ".read": true,
          ".write": true
        }
      }
    }
  }
}
```

**Варіант 2: Для тестування (небезпечно!)**

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

### 5. Збережіть правила
- Натисніть кнопку **"Publish"** (Опублікувати)
- Дочекайтеся підтвердження збереження

### 6. Перевірка
- Оновіть сторінку додатку (F5)
- Спробуйте додати автомобіль у режимі редагування
- Перевірте консоль браузера (F12) на наявність помилок
- Перевірте в Firebase Console, чи з'явилися дані в `/cars`

## Структура даних в Firebase

Дані зберігаються в такій структурі:

```
/cars
  /{carId}
    /brand: "Tesla"
    /model: "Model 3"
    /categories: [...]
```

## Якщо все ще не працює

1. Перевірте, чи правильно встановлено правила (має бути кнопка "Publish")
2. Перевірте консоль браузера на помилки (F12 → Console)
3. Перевірте, чи Firebase SDK завантажено (в консолі має бути "✅ Firebase ініціалізовано успішно")
4. Перевірте, чи правильний URL бази даних: `https://remcar-a23dc-default-rtdb.europe-west1.firebasedatabase.app`
